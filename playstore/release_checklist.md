# Tiny Next Google Play Release Checklist

This checklist is the source of truth for moving `com.batb4016.tinynext` from the local MVP branch to Google Play production. It separates local repo work from account-console work so future agents do not claim a Play launch before the console-only gates are actually complete.

## Local Release Readiness

- [x] Android package is `com.batb4016.tinynext`.
- [x] Target SDK is 35, which satisfies the current Play submission floor documented for new apps after August 31, 2025.
- [x] Version starts at `versionCode = 1` and `versionName = 1.0.0`.
- [x] Billing product ID in code is `remove_ads_premium`.
- [x] Store listing text drafts live in `playstore/`.
- [x] Public privacy policy source lives in `public/privacy/index.html` for Vercel hosting.
- [x] Final Vercel privacy URL is deployed at `https://tiny-next-vercel.vercel.app/privacy`.
- [ ] Final Vercel privacy URL is entered in Play Console.
- [x] Final Vercel privacy URL matches the default `https://tiny-next-vercel.vercel.app/privacy` before the final Play AAB is built.
- [ ] Production AdMob values are set through environment variables or local-only `local.properties`:
  - `ADMOB_APP_ID`
  - `ADMOB_BANNER_AD_UNIT_ID`
- [x] Local upload-key signing material was generated outside the repo under `C:\Users\hshre\.codex\secrets\tiny-next\`.
- [x] Upload-key signing values are available through the local-only env script `C:\Users\hshre\.codex\secrets\tiny-next\release-signing.env.ps1`:
  - `TINYNEXT_RELEASE_STORE_FILE`
  - `TINYNEXT_RELEASE_STORE_PASSWORD`
  - `TINYNEXT_RELEASE_KEY_ALIAS`
  - `TINYNEXT_RELEASE_KEY_PASSWORD`
- [x] Final signed release bundle is generated by `.\gradlew.bat clean test assembleDebug bundleRelease` with the signing env script loaded.
- [x] `jarsigner -verify -verbose -certs app\build\outputs\bundle\release\app-release.aab` confirms the AAB is signed.

## Manual Device QA

Use a fresh debug install before relying on the release bundle:

```powershell
adb -s R5CYA18JAXH uninstall com.batb4016.tinynext.debug
.\gradlew.bat :app:installDebug
```

Required checks:

- [ ] Onboarding first launch.
- [ ] Use sample tasks.
- [ ] Add Task with typed input.
- [ ] Edit Task with typed input.
- [ ] Pick next task.
- [ ] Another picks a different eligible task when possible.
- [ ] Done opens Stats, and Stats Back returns Home.
- [ ] Snooze removes the current task from the immediate picker pool.
- [ ] Delete task snackbar appears.
- [ ] Undo restores the deleted task.
- [ ] Settings Delete all local data clears local tasks/settings as designed.
- [ ] Premium screen loads without crashing when Play product details are unavailable.
- [ ] Crash buffer is clean after the QA pass.

## Play Console Setup

- [ ] Create the Play Console app as a free Android app.
- [ ] Use package name `com.batb4016.tinynext`; package names are permanent after upload.
- [ ] Enroll in Play App Signing.
- [ ] Use a Google-generated app signing key unless cross-store signature reuse is required.
- [ ] Use the local keystore only as the upload key.
- [ ] Create and activate the one-time in-app product `remove_ads_premium` at `$2.99`.
- [ ] Complete App access as "no login required."
- [ ] Complete Ads declaration as ads present.
- [ ] Complete Content Rating questionnaire.
- [ ] Complete Target audience and content section for a general productivity audience, not child-directed.
- [ ] Complete Data Safety using `playstore/data_safety_notes.md` as the working basis, with third-party SDK behavior reviewed against the final AAB.
- [ ] Enter the final public privacy policy URL.
- [ ] Upload title, short description, full description, screenshots, and feature graphic.
- [ ] Upload the signed AAB to Internal testing first.

## Personal Account Testing Gate

For a new personal Play developer account, production access requires a closed test with at least 12 opted-in testers for at least 14 continuous days.

- [ ] Create a closed testing track.
- [ ] Add at least 12 real tester Google accounts.
- [ ] Publish the internal-tested AAB to closed testing.
- [ ] Keep at least 12 testers opted in continuously for 14 days.
- [ ] Record tester feedback in `playstore/tester_feedback_log.md`.
- [ ] Apply for production access from the Play Console dashboard after the 14-day gate.
- [ ] In the production-access answers, cite the local/device QA evidence, tester feedback, local-only data model, banner-only ads, and one-time premium upgrade.

## Production Submission

- [ ] Promote the same closed-tested artifact or build a new signed AAB with an incremented `versionCode`.
- [ ] Submit for Play review.
- [ ] Monitor Play policy warnings, Android vitals, crash reports, billing setup, and AdMob serving status after approval.
